## 今日开发总结（2026-04-25）

### 1) 数据来源与初始化重构
- 阈值页移除硬编码演示阈值，改为从空数据启动。
- 项目配置在无部署数据时不再回退默认 demo，改为空配置（`visions/features` 都为空）。
- 阈值规则改为本地持久化（`localStorage`：`threshold_rules_store_v1`）。

### 2) 缺陷列表与命名修复
- 左侧缺陷列表只使用项目配置中的 `features`，不再出现无关默认项。
- 修复 `f2/f3` 与 i18n key 冲突导致显示 `Feature_2/Feature_3` 的问题：优先显示项目配置里的缺陷名称。

### 3) 阈值粒度能力扩展
- 阈值目标支持三种粒度：**视野 / CC / 图像**（ROI 保持单独逻辑）。
- “创建通用阈值”默认新增为**视野粒度**（例如先落在“正面”）。
- 行内目标改为单一下拉分组（视野/CC/图像），可直接切换粒度与目标，不再被固定在图像维度。

### 4) 筛选交互重做（联级筛选）
- 移除旧的“按图像/按视野”筛选入口。
- 新增三级联级筛选：
  1. 第一级：视野（含 ROI 入口）
  2. 第二级：CC（依赖第一级）
  3. 第三级：图像（依赖前两级）
- 修复从“视野”切到“CC/图像”后行被误过滤“消失”的问题。

### 5) 其他
- 同步更新中英文文案与样式（联级筛选标签、提示文案等）。
- 已进行语法与基础校验（`node --check` / lint 无报错）。
